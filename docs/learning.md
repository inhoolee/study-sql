# 학습에 대한 학습

## 학교 학습 VS 야생 학습

### 학교 학습

- 개별적/일방향
- 순차적 : 1페이지부터 시작, 논리적/연역적 흐름을 지킴
- 정적/명확한 목표 : 고정된 목표, 학습 목표가 구체적
- 제한된 자원 : 정해진 교과서, 시험 범위, 교사 등
- 단일 정답 : 하나의 문제에 하나의 답이 정해져있다고 믿음
- 통제된 환경
- 피드백이 드문 상황 : 피드백이 없거나 시험만 존재

### 야생 학습

- 협력적 : 학습자가 다른 사람들과 나누고 도움을 줌
- 비순차적 : 정보가 주어진 순서와 구조를 따를 필요가 없음. 현재 문제가 중요
- 필요와 흥미에 맞게 순서를 결정
- 동적/애매한 목표 : 지속적으로 목표를 조정해야 함
- 아웃컴 중심, 아웃컴을 내고 빠르게 경험
- 자원 : 무제한. 유튜브 영상 등
- 정답이 없는 경우도 있고, 복수 정답일 때도 있음
- 피드백이 빈번 : 중간 중간 피드백에 나타나게 됨(오류 메시지 발생)

### 도움이 된 학습

- 풀어야 하는 문제와 직결된 내용을 학습할 때
- 지금 배우는 내용이 왜 필요한가?를 잘 이해할 때
- 자기 주도적으로 생각해서 재구성할 때
- 결과물이 보이면서 피드백을 보고 더 개선할 수 있을 때
- 목표를 유연하게 가져갈 때
- 팀원, 동료들과 같이 진행할 때

## 학습 전략

1) 목표 정의 : 이걸 학습해서 어떤 것을 달성하기 위함일까? 구체적으로 정의
2) 그 분야를 잘 아는 사람이 있는가? : 있으면 자문 구하기
3) 현재 활용할 수 있는 교보재는? : 찾는데 시간 많이 쓰지않음. 어차피 모든 자료를 다 볼 수 없음
   1) 그 교보재를 내 목표에 맞게 재구성하려면?
   2) 일단 큰 그림을 한번 그리고, 자세히 가려면?
4) 제한된 시간 안에 학습(1시간, 4시간, 2일 등) : 난이도에 따라 다르게 설정
5) 중간 결과물을 만들고, 그것을 현업에서 어떻게 쓸지 생각
6) 더 정리할 시간이 있다면 템플릿처럼 정리(테라폼 등)
7) 글로 다시 한번 정리

## 우려되는 점

LLM의 발전으로 학습을 정말 빠르게 할 수 있는 시기
그러나 잘못 학습하면 스스로에게 남는 것이 없게 됨
LLM을 활용할 때, 직접적으로 답을 구하지 않고 힌트만 달라고 하는 것도 방법
